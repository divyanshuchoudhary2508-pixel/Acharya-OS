# Feature 38: IPv4

## What it does
IPv4 is the layer above Ethernet and ARP. It chooses a next hop, provides a checksum helper, and exposes a small route view for debugging.

## Current behavior
- local IP, gateway, and subnet are tracked in kernel state
- route lookup chooses direct delivery or the gateway
- `ipv4_send()` resolves the next hop through ARP and sends through Ethernet
- checksum helper exists for future packet construction
- `route` prints the current route view

## Notes
This is a first pass. It gives the stack a routing spine without pretending we already have protocol parsing for incoming packets.
